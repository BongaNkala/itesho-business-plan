Section 1

Introduction

1.1 Business Concept

Itesho is a project control platform developed by Thee Landlord (Pty) Ltd. The platform is designed to manage BOQ (Bill of Quantities), RFQs (Request for Quotations), compliance, and invoice control across multiple industries — with a primary focus on construction, but extending to any project-based environment.

The platform consists of a React + Vite frontend deployed on Vercel, and a Django REST Framework + PostgreSQL backend deployed on Render, with JWT authentication via djangorestframework-simplejwt.

The name "Itesho" reflects the platform's South African roots. The system is built with Rand-based pricing and local compliance requirements in mind, with a per-project, per-month pricing model.

💡 Core Purpose: Itesho exists to certify what gets invoiced — ensuring that only compliant, approved work is billed, reducing payment disputes across all industries.

1.2 Product Overview

  • Product Name: Itesho
  • Type: Cloud-based Project Control Platform (SaaS)
  • Industries: Construction, Infrastructure, Engineering, Facilities Management, Government Tenders
  • Core Modules: BOQ, RFQ, Compliance Gatekeeper, Daily Logs, PM Approval, Invoicing
  • Pricing Model: Per project, per month

Section 5

Products & Services

5.1 Itesho Platform Overview

Itesho is a project control platform built specifically for BOQ tracking, RFQ management, compliance, and invoice control.

5.1.1 Core Modules (Based on Live Application)

Dashboard:

  • System operational status (Performance: 98%, Active Projects: 1, Pending: 1)
  • Post-Award Project Control overview
  • Total Projects, Overall Progress, Total Budget, Pending Approvals
  • Recent Projects with progress and status

Projects:

  • Select a project to access (compliance verification required)
  • Project details: budget, spent, progress, timeline
  • Multiple projects per client

Compliance Gatekeeper:

  • Compliance Checklist with access control
  • Certificate: Safety File (approval required)
  • Liabilities: Insurance (upload required, approval required)
  • "All requirements met! You can now access this project."
  • Blocks access to daily logs if compliance is not met

RFQ (Request for Quotation) Management:

  • Create and manage RFQs for procurement
  • Track supplier responses
  • Compare quotations
  • Award contracts

Bill of Quantities (BOQ):

  • Hierarchical structure: Section → Subsection → Line Items (levels 1-3)
  • Planned Cost, Total Cost, Overall Progress tracking
  • Item details: ID, Description, Unit, Planned Qty, Rate, Approved Qty, Total Cost
  • Expandable sections for detailed BOQ breakdown

Daily Logs:

  • Record daily site/project activities, manpower, and progress
  • Select Project, Log Date
  • Tabs: Weather, Manpower, Equipment, Work, Safety, Quality, Delays, Photos
  • Save Draft and Submit functionality

Submissions:

  • All daily log submissions from contractors
  • Table: Date & Time, Project, Contractor, Work Description, Status, Action
  • Statuses: Approved, Pending
  • Full audit trail of all submissions

Invoices:

  • Auto-generated from approved Level 1 BOQ quantities
  • Total Planned, Claimable, Claimed, Remaining
  • Client details, itemised breakdown
  • Payment tracking

5.1.2 The Invoice Control Chain

The platform enforces a 4-step approval chain before an invoice is generated:

  1. Compliance Check — Gatekeeper verifies all compliance items
  2. Daily Log Submission — Site teams submit actual quantities
  3. PM Approval — Project Manager reviews and approves quantities
  4. Invoice Generation — System multiplies approved qty × rate
🔒 The Golden Rule: No compliance = No daily log = No approved quantities = No invoice.

5.2 Technical Architecture

  • Frontend: React + Vite, deployed on Vercel
  • Backend: Django REST Framework + PostgreSQL, deployed on Render
  • Auth: JWT (djangorestframework-simplejwt)
  • BOQ Hierarchy: Section → Subsection → Line Items (levels 1-3)
  • Key Libraries: django-cors-headers, python-dotenv

9.1 The Invoice Control Process

Itesho's core operational process ensures that only compliant, approved work gets invoiced:

  1. Compliance Verification: Gatekeeper checks safety permits, insurance, licences, inductions
  2. Daily Log Submission: Site teams submit actual quantities, photos, and notes

  4. Invoice Generation: Auto-calculated from approved quantities × rate
