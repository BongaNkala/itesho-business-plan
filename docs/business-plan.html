Section 1

Introduction

1.1 Business Concept

Itesho is a project control platform developed by Thee Landlord (Pty) Ltd. The platform is designed to manage BOQ (Bill of Quantities), RFQs (Request for Quotations), compliance, and invoice control across multiple industries — with a primary focus on construction, but extending to any project-based environment.

Itesho is installed software — deployed on the client's own infrastructure (on-premise or private cloud). This provides clients with full control over their data, security, and customisation.

The platform consists of a React + Vite frontend and a Django REST Framework + PostgreSQL backend, with JWT authentication via djangorestframework-simplejwt.

The name "Itesho" reflects the platform's South African roots. The system is built with Rand-based pricing and local compliance requirements in mind, with a once-off setup fee + monthly per-project pricing.

💡 Core Purpose: Itesho exists to certify what gets invoiced — ensuring that only compliant, Project Manager-approved work is billed, reducing payment disputes across all industries.

Section 6

Products & Services

6.1 Itesho Platform Overview

Itesho is a project control platform built specifically for BOQ tracking, RFQ management, compliance, and invoice control.

6.1.1 Core Modules (Based on Live Application)

Dashboard:

  • System operational status (Performance: 98%, Active Projects: 1, Pending: 1)
  • Post-Award Project Control overview
  • Total Projects, Overall Progress, Total Budget, Pending Approvals
  • Recent Projects with progress and status

Projects:

  • Select a project to access (compliance verification required)
  • Project details: budget, spent, progress, timeline
  • Multiple projects per client

Compliance Gatekeeper:

  • Compliance Checklist with access control
  • Certificate: Safety File (approval required)
  • Liabilities: Insurance (upload required, approval required)
  • "All requirements met! You can now access this project."
  • Blocks access to daily logs if compliance is not met

RFQ (Request for Quotation) Management:

  • Create and manage RFQs for procurement
  • Track supplier responses
  • Compare quotations
  • Award contracts

Bill of Quantities (BOQ):

  • Hierarchical structure: Section → Subsection → Line Items (levels 1-3)
  • Planned Cost, Total Cost, Overall Progress tracking
  • Item details: ID, Description, Unit, Planned Qty, Rate, Approved Qty, Total Cost
  • Expandable sections for detailed BOQ breakdown

Daily Logs:

  • Record daily site/project activities, manpower, and progress
  • Select Project, Log Date
  • Tabs: Weather, Manpower, Equipment, Work, Safety, Quality, Delays, Photos
  • Save Draft and Submit functionality

Submissions (Awaiting Project Manager Approval):

  • All daily log submissions from contractors
  • Table: Date & Time, Project, Contractor, Work Description, Status, Action
  • Statuses: Pending PM Approval, Approved, Rejected
  • Full audit trail of all submissions with approver identity

Invoices:

  • Auto-generated from Project Manager-approved Level 1 BOQ quantities
  • Total Planned, Claimable, Claimed, Remaining
  • Client details, itemised breakdown
  • Payment tracking
  • Only PM-approved work is included in invoices
Section 7

The Approval Chain

Itesho enforces a 4-step approval chain where Project Managers (or other authorised managerial roles) certify all work before it becomes invoiced.

🔒 1. Compliance
Gatekeeper
→
📋 2. Daily Log
Submitted by Site Team
→
✅ 3. Project Manager
Reviews & Approves
→
💰 4. Invoice
Auto-Generated
🔒 The Golden Rule: No compliance = No daily log = No Project Manager approval = No invoice.

7.1 Who Can Approve?

  • Project Manager — Primary approver for all project work
  • Construction Manager — Can approve work on behalf of the Project Manager
  • Site Manager — Can approve daily logs within their delegated authority
  • Quantity Surveyor — Can approve quantities and rates
  • Any authorised Managerial Role — Configurable based on client's organisational structure

Section 11

Operations Strategy

11.1 The Project Manager Approval Process

  1. Compliance Verification: Gatekeeper checks all compliance items
  2. Daily Log Submission: Site teams submit actual quantities
  3. Project Manager Review: PM reviews against site reality
  4. PM Approval: PM certifies the work — quantities are locked
  5. Invoice Generation: Auto-calculated from approved quantities

11.2 Installation & Deployment

  • Client Infrastructure: On-premise or private cloud
  • Supported OS: Ubuntu 20.04+, Windows Server 2019+
  • Database: PostgreSQL 13+
  • Setup Time: 2-5 days depending on complexity
